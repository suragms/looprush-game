## LOOP RUSH — Full Build Plan (Phases 1–4)

Frontend-only neon arcade survival game. React + TypeScript + Vite + Phaser 3 + Tailwind + Zustand. All state in `localStorage`. Mobile dash = on-screen **DASH button + tap-to-dash** (drag is movement). Built phase-by-phase; each phase ends with `npm run lint && typecheck && test && build` all passing.

### Architecture decisions
- **React vs Phaser split:** React renders full-screen menus only (Home, Result, Settings, Achievements, Stats, Customize, Daily). Phaser owns the canvas game loop **and** the in-game HUD (score/combo/level/dash/powerup) as a parallel `HUDScene` → **zero React re-renders during gameplay.** React menus unmount while playing.
- **Bridge:** a thin Zustand store (`useGameStore`) for screen routing + a `gameOver` callback the Phaser scene calls to flip React to the Result screen. Settings/stats also live in Zustand, hydrated from `localStorage` on boot.
- **Perf:** object pooling for obstacles/energy/particles, no per-frame allocations, `visibilitychange`/`blur` auto-pause, hard entity caps, LOW/MED/HIGH quality tiers + adaptive FPS monitor that suggests LOW.
- **Storage:** single versioned key `loopRush_save_v1` with strict validation + migration; defaults restored on corruption; writes only on meaningful changes (never per frame).

### Tech & exact tooling
- Vite 5, React 18, TypeScript 5 (strict), Phaser 3.80, Tailwind 3, Zustand 4, Vitest (tests), ESLint + Prettier, `vite-plugin-pwa` (PWA/offline).
- Scripts: `dev`, `build`, `preview`, `lint`, `typecheck` (`tsc --noEmit`), `test` (vitest run), `test:watch`.

### Phase 1 — Core playable slice
- Scaffold Vite project, install deps, configure TS strict + ESLint + Prettier + Tailwind + Vitest + path aliases.
- Boot/Preload scenes generating neon orb/energy/obstacle graphics procedurally (no asset pipeline needed).
- `GameScene`: arena, Player orb (WASD/arrows + drag + optional mouse-follow), Dash (Space/button/tap; cooldown, invuln, trail), Energy pickup, **3 obstacles** (BasicChaser, Bouncer, Spinner), collisions, screen shake, particles.
- `Combo`/`Scoring`/`Difficulty` systems: every 30s → LEVEL n; combo x1→x20 with decay; near-miss detection.
- `HUDScene`: score, combo (animated), level, dash cooldown ring, pause.
- React: Home (logo, best score, PLAY, Settings), Result (score/best/new-best/time/combo/energy/misses/level + Play Again/Home/Share), Settings (audio toggles/volume/shake/motion/contrast/quality).
- SaveManager: load/validate/save high score + settings + stats.
- Auto-pause on tab hide. Instant restart (<1s).
- **Tests:** combo, scoring, difficulty progression, storage load/corruption recovery.

### Phase 2 — Gameplay depth
- Remaining obstacles: SpeedChaser, Laser, Meteor, Splitter, Hunter — gradual introduction gated by level.
- **Random events** (20–40s, with warning telegraphs, always survivable): MeteorRain, Blackout, EnergyStorm, SpeedSurge, ShrinkZone, GravityShift, LaserGrid, GoldRush.
- **Power-ups** (random spawns, duration UI): Shield, Magnet, SlowTime, GhostMode, ScoreBoost, EnergyBlast.
- **AudioManager** (WebAudio-synthesized SFX + music, no asset files): collection/combo/dash/nearmiss/powerup/collision/gameover + music/SFX toggles + volume. No autoplay before interaction.
- Visual polish: glow, particle bursts, combo milestone animations, animated score.

### Phase 3 — Retention
- **Achievements** (FIRST RUN, SURVIVOR, UNTOUCHABLE, COMBO MASTER, DASH MASTER, ENERGY HUNTER, VETERAN…) with non-blocking toasts; persisted.
- **Statistics** screen (all tracked stats).
- **Daily challenges** — deterministic from local date via seeded RNG; same date → same challenge; completion persisted; no network.
- **Cosmetics** — orb colors/shapes, trails, dash effects, arena themes; unlocked via score/achievements/games/daily/level; equip + preview in Customize screen; cosmetics are visual only.

### Phase 4 — Quality
- **PWA**: `vite-plugin-pwa` (manifest, generated icons, splash), service worker offline caching; playable offline after first load; installable.
- **Accessibility**: reduced motion, screen-shake toggle, high-contrast mode, color-independent hazard shapes/icons, full keyboard control, visible focus in menus.
- **Graphics settings** LOW/MED/HIGH (particle/glow budgets) + adaptive auto-suggest LOW on sustained FPS drop.
- **Performance** hardening: pooling audits, entity caps, pause-on-hidden, resize/orientation handling, `dvh`/safe-area units.
- **Tests** expansion: achievements unlock, daily challenge determinism, storage migration, share util.
- **Share**: Web Share API + clipboard fallback (score/combo/level/challenge text).
- Final `lint && typecheck && test && build` clean; dev server smoke check.
